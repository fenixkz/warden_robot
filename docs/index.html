Warden_robot’s documentation!

This is the documentation of the warden_robot package! This package is the solution for the first assignment of Experimental Robotics class of the University of Genoa (UniGe). In order to get to know the assignment better, please refer to https://www.github.com/fenixkz/warden_robot

State Machine module

This script is the main script that creates the state machine based on SMACH ROS library. The necessary connections to ontology via armor_py_api is done via the helper class state_helper.ProtegeHelper Interface helper (state_helper.interface_helper) is used to do the logic described by the surveillance policy

build_map()[source]

Method that builds the topological map of the environment. First, it invokes state_helper.ProtegeHelper.load_topological_map and state_helper.ProtegeHelper.add_individuals. Also, adds the dataproperty ‘visitedAt’ to each location with the now timestamp. Finally, calls the state_helper.ProtegeHelper.reason.

Returns

Error code. 1 for success. 0 for error.

Return type

int

decide_next_location()[source]

Method that uses the surveillance policy to decide what should be the next location of the robot. First, it calls state_helper.ProtegeHelper.get_urgent_location to know if there are urgent locations, and state_helper.ProtegeHelper.canReach to know the reachable locations. If no locations became urgent then returns a random reachable location. Else chooses random between the intersection of reachable and urgent locations.

Returns

String representation of the next location.

Return type

str
